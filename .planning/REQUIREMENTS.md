# Requirements: Le Commun

**Defined:** 2026-03-08
**Core Value:** La balance nette entre Chris et Alex est toujours visible et juste — report automatique, zero saisie manuelle, zero erreur de copie.

## v1.4 Requirements

Requirements for milestone v1.4 — Récurrences. Each maps to roadmap phases.

### Récurrence

- [ ] **REC-01**: User peut marquer une dépense comme récurrente via un toggle dans le formulaire
- [ ] **REC-02**: User peut marquer un ajustement comme récurrent via un toggle dans le formulaire
- [ ] **REC-03**: User peut désactiver la récurrence sur une dépense existante
- [ ] **REC-04**: User peut désactiver la récurrence sur un ajustement existant
- [ ] **REC-05**: Les items récurrents sont visuellement identifiables dans les listes

### Report automatique

- [ ] **RPT-01**: À la création d'un nouveau mois, les dépenses récurrentes actives du mois précédent sont copiées automatiquement
- [ ] **RPT-02**: À la création d'un nouveau mois, les ajustements récurrents actifs du mois précédent sont copiées automatiquement
- [ ] **RPT-03**: Les items reportés sont des copies indépendantes (modifiables et supprimables)
- [ ] **RPT-04**: Les items reportés héritent du flag récurrent de la source

## Future Requirements

None deferred for this milestone.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Fréquence personnalisable (hebdo, trimestriel) | Complexité inutile — mensuel couvre 100% des cas |
| Historique des récurrences | Pas de besoin identifié |
| Récurrences avec date de fin | Toggle on/off suffit |
