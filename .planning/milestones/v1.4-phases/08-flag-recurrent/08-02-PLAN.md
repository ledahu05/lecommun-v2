**DepenseForm.tsx:**
Add a recurrent toggle inside the form, placed BEFORE the submit button. Use a simple checkbox approach with a `<label>` wrapping an `<input type="checkbox" name="recurrent">` styled as a toggle-like control. Use the Repeat icon from lucide-react for visual clarity.

Implementation:
1. Import `Repeat` from `lucide-react`
2. Add state: `const [recurrent, setRecurrent] = useState(false);`
3. Add this block before the error display and submit button:
```tsx
<div className="flex items-center gap-3 py-2">
  <label className="flex items-center gap-2 cursor-pointer select-none">
    <input
      type="checkbox"
      name="recurrent"
      checked={recurrent}
      onChange={(e) => setRecurrent(e.target.checked)}
      className="sr-only peer"
    />
    <div className="w-10 h-6 bg-muted rounded-full peer-checked:bg-primary relative transition-colors">
      <div className="absolute left-0.5 top-0.5 w-5 h-5 bg-white rounded-full shadow transition-transform peer-checked:translate-x-4" />
    </div>
    <Repeat className="h-4 w-4 text-muted-foreground" />
    <span className="text-base">Recurrent</span>
  </label>
</div>
```

IMPORTANT: The sr-only + peer approach may not work across the label boundary. Use a simpler approach instead -- a Button toggle that sets a hidden input:
```tsx
<div className="flex items-center gap-3">
  <input type="hidden" name="recurrent" value={recurrent ? 'on' : ''} />
  <Button
    type="button"
    variant={recurrent ? 'default' : 'outline'}
    className="min-h-[48px] text-base gap-2"
    onClick={() => setRecurrent(!recurrent)}
  >
    <Repeat className="h-4 w-4" />
    Recurrent
  </Button>
</div>
```

This follows the existing pattern in the form (segmented buttons for paye_par, categorie). Place this after the optional fields section (date/label) and before the error/submit.

**AjustementForm.tsx:**
Same pattern. Add `recurrent` state, `Repeat` import, and the toggle button before the error/submit section. Add `<input type="hidden" name="recurrent" value={recurrent ? 'on' : ''} />`.

Mobile-first: Button is min-h-[48px] per CLAUDE.md rules. Text is text-base (16px minimum).
