<decisions_made>

- Used native DOM event listener (`useEffect` + `addEventListener('change', ...)`) instead of React's `onChange` for file input — Playwright's `setInputFiles` doesn't reliably trigger React's synthetic onChange in Next.js 15/React 19
- Added `data-ready="true"` attribute to file input after event listener is attached — allows tests to wait for hydration before calling `setInputFiles`
- Used `router.refresh()` after server actions (import + delete) — `revalidatePath` alone doesn't cause immediate client re-render when action called via `useTransition`
- Used `AlertDialog` from `radix-ui` unified package for delete confirmation (matching existing button.tsx pattern)
- MoisCard restructured: `<div data-testid="mois-card">` wrapping `<Link>` + `<DeleteMoisButton>` — avoids nested `<a>` tags
- Tests use `.filter({ hasText: 'janvier 2026' })` instead of `.first()` — `loginAs` auto-creates mars 2026 via dashboard navigation
